The Crisis

The Skills Gap is a Data Problem

The Fourth Industrial Revolution is rewriting every job description. AI, cloud-native architectures, automation, and data-driven decision-making demand skills that did not exist five years ago. Yet most organisations still measure workforce readiness with annual reviews, static job descriptions, and gut-feel succession planning.

  • 85 million jobs displaced and 97 million new ones created by 2025 (WEF estimate)
  • Half of all employees need reskilling within five years to stay employable
  • HR departments lack the data infrastructure to spot gaps before they become crises
  • Coaches and managers operate blind — without anonymised, real-time skill telemetry
Central Thesis

HR as Real-Time Telemetry

CareerForge treats workforce readiness the way modern observability platforms treat infrastructure: continuous instrumentation, dimensional scoring, automated alerting, and AI-co-produced recommendations.

Stop reviewing employees once a year. Start measuring readiness continuously, scoring it dimensionally, and acting on it algorithmically. The CareerForge thesis

Three roles, three portals, one anonymised data plane — with a manager overlay for team accountability.

Frontend — Vue 3 + Vite + pnpm

  • Coach portal — catalogue, advice, forms, audit
  • Admin portal — users, taxonomy, connectors, readiness
  • Bauhaus design system — sharp corners, primary colours
Intelligence Layer

The RAG Pipeline

  • Connector ingestion — 11 templates (Perplexity, Anthropic, Stack Overflow, Hacker News, GitHub Trending, Medium, LinkedIn, Indeed, DEV.to, RSS, custom REST). Celery beat schedules pulls.
  • Document parsing — RSS/Atom XML, JSON, HTML normalised into chunks with provenance metadata.
  • Embedding — Ollama running nomic-embed-text (768 dims) inside the cluster. No data leaves the perimeter.
  • Vector store — pgvector inside the same Postgres instance. One less moving part.
  • Generation — Perplexity AI (sonar-pro) for synthesis. Responses cached by topic with TTLs from 6 hours to 7 days.

Every AI suggestion is co-produced — grounded in real connector documents, never hallucinated.

Latest Feature

Putting Managers in the Loop

A development plan needs three signatures: the employee commits to it, the coach validates the plan, and the manager signs off on the team-level commitment. CareerForge models this explicitly.

  • JWT-driven gate — the manager realm role unlocks the "My Team" tab in the Employee portal
  • Six new endpoints — team list, analytics, commitments, pending IDPs, sign-off, bench activity
  • Org hierarchy — 5 managers with 6 to 35 direct reports each, modelled via manager_pseudo_id
  • IDP workflow — awaiting employee → awaiting coach → awaiting manager → signed

Delivery Model

100% GitOps, Zero ClickOps

  • ArgoCD app-of-apps — one root Application materialises 15 child apps from a single Gitea repo
  • Kustomize for backend — dev overlay patches replicas, uvicorn loop, secrets injection
  • Helm for frontends — one cf-portal chart, three values files, three releases
  • Self-healing — selfHeal=true means manual kubectl edit is reverted within seconds
  • Local registry — images live in self-hosted Gitea. No external dependencies, fully offline-capable

Reproducible from a clean machine in under twenty minutes via a single bootstrap script.

War Stories

What We Learned the Hard Way

  • SQLModel vs SQLAlchemy — session.exec() breaks greenlet context under uvloop. Fix: raw session.execute().scalars(), NullPool, lazy engine creation.
  • k3d image imports — multi-arch manifests fail silently. Fix: docker save piped to ctr images import on each node.
  • CoreDNS NodeHosts wipe — every k3d cluster start resets hosts on the nodes. Fix: re-patch CoreDNS configmap on every restart.
  • crypto.randomUUID() over HTTP — undefined in non-secure contexts. Fix: fallback to crypto.getRandomValues.
  • Bitnami images vanished — only SHA-tagged remain on Docker Hub. Fix: mirror locally in Gitea registry.
Closing

Three Things to Remember

  • HR is a data platform problem. Organisations that win the Industry 4.0 transition will treat workforce readiness with the same telemetric rigour they bring to production systems.
  • Privacy and personalisation are not enemies. Pseudonymous IDs and consent tokens let coaches help without surveilling. The schema decides.
  • GitOps scales because discipline scales. Click-driven environments accumulate technical debt invisibly. Git-driven environments accumulate it visibly — and visible debt is debt you can pay.

Build for the workforce you will have, not the one you have today.
