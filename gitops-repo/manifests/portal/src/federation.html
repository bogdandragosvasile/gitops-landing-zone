Claude Agent Federation

A 6-agent, depth-2 hierarchy that manages the base landing zone. Agents coordinate via a shared state file, dispatch work in parallel phases, and use 12 reusable skills for common operations. Extend this federation with your own domain agents when you add applications.

Agent Hierarchy

orchestrator (depth 0, opus) ├── platform-infra (depth 1, opus) │ ├── cluster-lifecycle (depth 2, sonnet) — k3d, network, metallb, coredns │ └── gitea-argocd (depth 2, sonnet) — gitea, argocd, keycloak, oidc │ ├── observability (depth 1, sonnet) — prometheus, grafana, loki, promtail │ └── security (depth 1, sonnet) — sealed-secrets, vaultwarden, rbac, netpol

6 agents total: 1 orchestrator + 3 domain agents + 2 sub-agents. Max 6 concurrent threads per agent.

Extending the Federation

When you add an application to this landing zone, create a new depth-1 agent for it and optionally add depth-2 sub-agents. Drop a new file in .claude/agents/your-app.md. Follow the pattern used in platform-infra.md — declare responsibilities, owned paths, owned namespaces, and coordination rules.

State Coordination

Agents share state via .claude/team-state.json (gitignored). Before starting work, each agent reads the file to check readiness flags. After a task completes, the agent updates its status and appends to the history array. Cross-domain dependencies (e.g., cluster_ready, argocd_ready) gate execution order.
